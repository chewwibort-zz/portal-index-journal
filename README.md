# portal-index-journal

一个用于归档和发布独立 HTML 页面的仓库。

## 项目简介

本仓库用于收集、归档和发布多个彼此独立的 HTML 页面。这些页面可能包含信息展示、工具页面、说明文档或其他静态内容，不针对任何特定域名或网站。

所有页面均为纯前端 HTML 文件，不依赖后端服务，可直接在浏览器中打开或通过 GitHub Pages 等服务进行托管。

## 目录说明

```
portal-index-journal/
├── index.html          # 仓库根目录的入口页面（可选）
├── pages/              # 存放主要 HTML 页面文件的目录
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
├── assets/             # 存放页面所需的静态资源（图片、样式、脚本等）
│   ├── css/
│   ├── js/
│   └── images/
├── archive/            # 存放已归档的旧版本或不再更新的页面
└── README.md           # 本文件
```

## 页面归档说明

- 每个 HTML 页面应尽量保持独立，避免跨页面依赖。
- 建议为每个页面编写简短的说明或元数据（可在页面内或通过外部文件记录）。
- 已不再维护或需要保留历史版本的页面可移入 `archive/` 目录，并在文件名中标注版本或日期（例如 `page-001-v1.html`）。
- 仓库不承诺持续更新所有页面，部分页面可能仅作为存档存在。

## 维护说明

- 欢迎通过 Pull Request 提交新的独立 HTML 页面，或对现有页面进行改进。
- 提交的页面应遵循基本的 HTML 规范，内容清晰，不包含恶意代码。
- 不建议在页面中嵌入外部跟踪脚本、广告或与页面内容无关的第三方资源。
- 本仓库的维护者会定期检查页面可访问性，但不对页面内容的准确性或时效性做出保证。

## 其他

- 本仓库仅用于归档和展示静态 HTML 页面，不提供任何形式的服务或接口。
- 如有疑问或建议，请通过 Issues 进行讨论。
